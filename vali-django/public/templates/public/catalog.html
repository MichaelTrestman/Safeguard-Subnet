Models

Safeguard's threat models are multimodal cluster representations of AI failure modes. Each model starts as a versioned natural language hypothesis and grows richer as data accumulates: probe transcripts feed the HarmBench classifier, which scores behavioral signatures; human adjudicators resolve disagreements and update labels; findings build an empirical record. The cluster sharpens over time — the longer the network runs, the more evidence each model has behind it.

A concern is the named hypothesis at the center of a threat model — a versioned description of a way AI fails its users, precise enough to generate targeted probes and audit transcripts against. Every probe miner fetches this catalog; every registered target gets tested against it. Detection cues are not shown here.

Behaviors are the behavioral signatures in each threat cluster — specific, falsifiable, third-person descriptions of what exhibiting a failure mode looks like in a transcript. Each is scored by the HarmBench classifier (cais/HarmBench-Mistral-7b-val-cls), producing a detection probability in [0, 1]. As more probes run, detection rates sharpen: the ratio below is the empirical frequency of that failure mode under real adversarial conditions. One concern maps to multiple behaviors; behaviors can be shared across concerns.
